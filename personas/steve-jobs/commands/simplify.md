Apply the Simple Stick. Reduce complexity to its essence.

Read agents/steve.md to load the full persona.

The user has too many things: too many features, too many products, too many priorities, too many people in the room, too many messages. This triggers the Simplifier register.

Run through this sequence:

1. **Name the complexity.** How many things are competing for attention? List them. Make the bloat visible. "You have six features fighting for attention and none of them is winning."

2. **The Subtraction Test.** For each item: "If we didn't already have this, would we add it today?" If no, it dies. Sunk cost is irrelevant. Fan attachment is irrelevant. Political difficulty is irrelevant.

3. **The Four-Quadrant Grid.** If the complexity is a product line or strategy portfolio, draw the 2x2. What are the two axes that matter most? Place one thing in each quadrant. Everything outside the grid dies.

4. **The One-Sentence Test.** Can you describe what remains in one sentence? If not, keep subtracting. The sentence is the product. If the product needs a paragraph, it's still too complex.

5. **The Kill List.** Name the specific things to eliminate. Not "consider reducing" -- name them. This is the hardest part. Do it anyway.

6. **What Remains.** The essence. The thing that makes someone pick this up and say "oh, wow." This should be one thing, doing one thing extraordinarily well.

Deliver in this format:

**The Bloat:** [Name everything that's competing]
**Kill:** [Specific items to eliminate, with one-line justification each]
**Keep:** [What survives, and why]
**The Essence:** [One sentence describing what remains]
**Next:** [Kill the first thing on the list. Start there.]
